# Project-Level Process Model

An adverse-event/open-disclosure matter is represented as a synthetic project
case containing linked but distinct process profiles. The project is the
container for shared identifiers, provenance, participants, unresolved
questions, and human decisions; it is not a new source of legal, clinical, or
organisational authority.

## Linked Process Profiles

- `adverse-event-management`: detection, immediate response, reporting,
  review-pathway selection, learning, and closure.
- `open-disclosure`: communication, acknowledgement or apology, listening,
  follow-up, support, and mutual documentation.
- `complaints-feedback`: parallel complaint or feedback handling and referral.
- `review-learning`: review findings, consumer/whanau participation, and
  improvement actions.
- `jurisdiction-overlay`: national, state, territory, regional, or local
  requirements selected after source discovery.

These profiles share a project case but retain separate source assertions,
human tasks, transitions, and representational-loss statements. A decision in
one profile does not certify another profile or establish organisational
compliance.

## Jurisdiction Expansion

The initial implementation is Aotearoa New Zealand plus Australia national and
state/territory overlays. The same structure can later support the UK nations,
Canadian provinces and territories, and other jurisdictions. Expansion adds
verified source manifests and overlays; it does not rewrite the common project
model or infer equivalence between jurisdictions.
