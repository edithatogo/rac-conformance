# pic-crosswalk 0.1.0

`pic-crosswalk` maps policy concepts, variables, parameters, decisions, evidence items, and process steps across concrete systems. It is a table format, not a universal ontology. Rows are added only when there is a named consumer or converter.

## File Shape

Each crosswalk file is a JSON object:

```json
{
  "conformsTo": "pic-crosswalk/0.1.0",
  "jurisdictionScope": {"country": "NZ", "domain": "official-information"},
  "provenance": {"curator": "Dylan", "method": "ai-proposed", "source": "Track 1 sample"},
  "rows": []
}
```

## Row Shape

Rows have stable package-scoped IDs and one or more system mappings:

```json
{
  "id": "nz-oia/parameter.working_day_limit",
  "label": "OIA maximum response time",
  "kind": "parameter",
  "dataType": "duration_working_days",
  "period": "case",
  "mappings": [
    {"system": "foi-o", "ref": "working_day_limit", "method": "ai-proposed"}
  ],
  "sourceRefs": ["Official Information Act 1982 s 15"],
  "definition": "Maximum working-day limit for a response decision."
}
```

## Methods

`method` is one of:

- `human-approved`: curated or verified by a human accountable for the interpretation.
- `ai-proposed`: drafted by an AI agent and not yet independently verified.
- `mechanical`: generated by deterministic tooling from a documented source.

Consumers must be able to filter out `ai-proposed` rows.

## Kinds

`kind` is one of `concept`, `variable`, `parameter`, `decision`, `evidence`, or `process_step`.
